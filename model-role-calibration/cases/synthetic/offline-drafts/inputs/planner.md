# 需求：移动端离线草稿同步

移动端表单需要支持离线编辑。用户恢复网络后，草稿自动同步到服务端。

已知条件：

- 一个用户可能在两台设备编辑同一份草稿。
- 服务端记录包含 `draft_id`、`revision`、`updated_at` 和正文。
- 客户端当前只有本地正文和最后保存时间，没有同步状态。
- 提交接口支持客户端提供幂等键，但尚未定义生成规则。
- 同步失败不能丢失本地编辑；用户必须能继续离线编辑。
- 第一版不支持多人实时协同，也不要求字段级自动合并。
- 本地数据库 schema 将从 v1 升级到 v2。

请给出可直接实施的计划，并明确无法自动解决的冲突如何交给用户处理。
