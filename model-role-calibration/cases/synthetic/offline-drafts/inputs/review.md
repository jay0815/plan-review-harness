# 需求背景

移动端表单支持离线草稿。一个用户可能在两台设备编辑同一草稿。服务端有 `revision`，提交接口支持幂等键。同步失败不能丢失本地编辑，用户仍可离线继续。第一版不做实时协同或字段级自动合并；无法自动解决的冲突必须交给用户处理。本地数据库从 v1 升到 v2。

# 待审查方案

1. 本地数据库 v2 新增 `synced: boolean`。
2. 用户每次编辑后保存正文并设置 `synced=false`。
3. 网络恢复后上传所有 `synced=false` 的草稿；成功后设置 `synced=true`。
4. 服务端始终接受最后到达的正文，并把 `revision + 1`。
5. 幂等键使用当前时间戳生成；每次重试重新生成。
6. 批量同步时只要有一条失败就停止；已经成功的记录保持服务端状态，本地统一保持 `synced=false`，下次全部重试。
7. v1 升级到 v2 时，为所有现有草稿设置 `synced=true`。
8. 验收方式：离线修改一份草稿，联网后能在服务端看到相同正文。
