# 需求背景

本地 CLI 管理插件文本资产，支持 user/project scope、dry-run、幂等更新和安全卸载。用户可能修改文件，目标目录也可能存在其他来源资产。

# Architecture Reviewer

- CLI 不应承担插件安装职责，所有安装和分发都必须交给在线 marketplace。
- 删除整个目标目录破坏资产所有权边界，CLI 必须记录自己管理的文件。
- user 和 project scope 应共享同一套路径解析与生命周期逻辑。

# Execution Reviewer

- 缺少 manifest 的格式、写入时机、版本字段和文件 hash，无法实现状态检查与安全更新。
- `--dry-run` 没有输出逐文件动作，不能用于预检。
- 失败后让用户重跑不构成恢复策略，需要临时目录、原子替换或备份恢复。

# Risk Reviewer

- 直接覆盖会丢失用户修改，整目录删除还可能删除其他来源文件。
- 为防止更新事故，必须建设灰度发布平台和远程 kill switch。
- 空目录 happy path 无法覆盖重复执行、部分失败、用户修改、scope 隔离和未知文件保留。

# 合成任务

合并重复问题，识别真正的方向分歧，降权无证据或超范围建议，并给出修订指令。
