# 需求：本地 CLI 管理插件资产生命周期

一个本地 CLI 需要把随版本发布的插件资产安装到用户目录或项目目录，并支持更新、卸载和检查状态。

已知条件：

- 插件由若干 Markdown、JSON 和模板文件组成，不包含可执行二进制。
- 用户可能会手工修改已安装文件。
- 同一目录中还可能存在其他工具或用户自己创建的文件。
- CLI 必须支持 `--scope user|project` 和 `--dry-run`。
- 安装、更新和卸载需要可重复执行，失败后可以恢复。
- CLI 只能管理自己安装的资产，不能删除无法确认归属的文件。
- 第一版不建设在线 marketplace、账号系统、后台服务或自动更新守护进程。

请给出可直接实施的计划。
